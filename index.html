CC3200 Lab Project Demo

Project Description

This is a demo website showcasing the CC3200 Lab Project. The project involves developing various interactive games and demos using the CC3200 microcontroller. Users can navigate through the different games and enjoy the interactive experiences provided by each one.

Login

Users enter their username and password using the IR remote control with numbers. T9-style text messaging is used to input the required information.

Main Menu

After successful login, the screen switches to a main menu page where users can select different games by pressing specific buttons on the remote control.

Tic Tac Toe

Two players take turns playing Tic Tac Toe using the remote control to select the position to place Xs or Os. Button 0 is used to move to the next square, and mute (or enter) is used to place the X or O. The game determines the winner and displays the result before returning to the main menu.

Number Memory

Users are shown a number on the screen and have to memorize it. Pressing the SW3 switch starts the game. The screen turns black, and users must enter the previously shown number. If the guess is correct, the game adds another number to memorize and increases the score. If the guess is incorrect, the game ends, and the score is sent through AWS, triggering an email.

Word Memory

Similar to Number Memory, users are shown words and must remember whether they have seen them before. In addition to keeping score, users have three lives. The SW3 and SW2 switches are used to indicate new and seen words, respectively. When users run out of lives, the score is sent through AWS.

Battleship

Two players can play Battleship on a 4x4 grid due to the OLED display's size limitation. Each player has their own OLED display to place ships and track attacks. UART communication is used between the LaunchPads to send messages. Attacks are visually indicated with red or cyan circles on both players' displays. Once all ships are sunk, a message is sent to the opponent via UART, and the screens display unique messages based on the outcome. A message is also sent through AWS.
